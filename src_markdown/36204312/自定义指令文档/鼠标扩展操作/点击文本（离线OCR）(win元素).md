# 描述
通过离线(免费)OCR识别在win元素上识别文字位置，然后执行点击操作

![](https://cdn.nlark.com/yuque/0/2023/png/22242030/1702620920492-11f246a8-c390-4318-bcd1-8e5279bb1c22.png)

# 参数配置
## 指令输入
### 常规
**元素对象**：win元素对象，需要进行OCR识别的win元素对象

**点击目标**：字符串，需要点击的文字或者正则表达式

**匹配方式**：下拉列表，匹配目标的方式

+ 包含：只要包含即可
+ 正则：通过正则表达式，兼容性会更好
+ 等于：完全相等

**使用缓存数据**：复选框，

+ 勾选：使用上次识别的结果
+ 不勾选：重新生成截图并进行OCR识别





### 高级
**识别区域**：下拉列表，如果大概知道文字位置，可以限制一下识别区域，加快查找速度。

+ 全部
+ 右上角(第一象限)
+ 左上角(第二象限)
+ 左下角(第三象限)
+ 左下角(第四象限)
+ 中间：中间区域的大小和象限大小一致
+ 左半区
+ 上半区
+ 右半区
+ 下半区

**点击识别结果的**：下拉列表，如果匹配到多个识别结果，选择点击的目标

+ 第一个
+ 最后一个
+ 全部

**执行后延迟(秒)**：数字，点击后的延时时间，如果点击多个目标，则每点击一个目标延迟一次

**横向偏移：**正数向右，负数向左

**纵向偏移**：正数向下，负数向上



其他参数和鼠标点击的参数功能一致



## 指令输出
无

# 注意事项
图片的区域以及图片上文字的数量都会影响到OCR识别的速度，所以如果想要提高速度，建议合理利用**缓存**参数以及**识别区域**参数
